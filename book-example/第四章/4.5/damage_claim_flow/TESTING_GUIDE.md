# 🧪 破损理赔智能助手 — 测试用例与预期结果

本文件包含 5 个典型测试场景，用于验证 **LangGraph 主流程、子图并行、业务逻辑、人工审批（Interrupt）、最终生成说明** 等功能是否正常运行。

---

## ✅ 场景 1：普通易碎品，风险低（自动通过）

### 输入示例

- 订单号：`ORDER-001`
- 用户 ID：`USER-001`
- 金额：`188`
- 品类：`fragile`
- 描述：`到货时玻璃茶壶破碎`
- 历史理赔总数：`0`
- 最近 30 天理赔：`0`
- 历史订单数：`20`

### 预期结果

- 规则检查：`eligible`（易碎品 + 时效内）
- 风险评估：`low`
- 相似案例：匹配 `fragile` 的历史案例（约 2 条）
- 人工审批：**不需要**（金额 < 300 且风险为 low）
- 最终说明：模型自动生成“全额退款”的友好话术，例如：
  > 经核实，您的订单在运输过程中出现破损，我们已为您办理全额退款……
- Streamlit 右侧显示：
  - 顶部：✅ “本次理赔流程已完成”
  - 下方：最终结果说明 + 完整 State JSON
