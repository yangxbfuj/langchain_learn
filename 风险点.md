# 技术性风险
1. (重要)HTTPS/WSS 需要TSL加密的场景，能做到加密通信，但是内网自签名证书无法校验证书有效性，不能杜绝中间人攻击
2. 诱骗设备UDP通信是明文， 且未做任何加密
3. (重要)B/S 架构安全规范中， 基本只描述了测试方法， 有没有比较具体得测试用例？
4. 部分网络安全应该有响应网络工程师保证， 如划分网络VLAN等，做好安全隔离
